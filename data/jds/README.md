# Job Descriptions for Evaluation (Track A)

## Purpose
These are NEW job descriptions — roles Olga has NOT applied to and has no outcome data for.
Used for Track A: unbiased relevance labelling (0–3 scale).

## Naming convention
```
{company}_{role_short}.txt
```
Examples:
- `google_people_data_scientist.txt`
- `notion_assessment_specialist.txt`
- `figma_ux_designer.txt`   ← out-of-domain example

## Target composition (25–30 files total)

| Type | Count | Examples |
|------|-------|---------|
| Strong fit — people data science, psychometrics | ~10 | People Scientist, Assessment Scientist |
| Borderline — adjacent (data analyst, research) | ~10 | Research Analyst, Insights Manager |
| Out-of-domain — clearly irrelevant | ~5 | UX Designer, Software Engineer, Marketing |

## How to add a JD
1. Copy the full job description text
2. Save as plain `.txt` file with the naming convention above
3. Add entry to `labels.json` (see evaluation/ folder)
